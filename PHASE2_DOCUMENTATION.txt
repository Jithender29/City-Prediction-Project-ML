SECTION 5: HANDLE MISSING VALUES (COUNTRY-WISE MEAN IMPUTATION)
────────────────────────────────────────────────────────────────────────────────
Purpose: Fill missing values in sparse features using domain-informed strategy

Strategy: Country-wise Mean Imputation (Two-level Approach)

LEVEL 1: Country-wise Mean
  For each sparse feature:
    1. Group data by Country
    2. Calculate mean of feature for each country
    3. Fill NaN values with the respective country's mean
    
  Rationale:
    - Cities in the same country often have similar characteristics
    - Preserves geographical/cultural patterns in the data
    - More informed than global mean (local context matters)

LEVEL 2: Global Mean (Fallback)
  For any remaining NaN values after Level 1:
    1. Calculate global mean across all cities
    2. Fill remaining NaN with this global value
    
  Purpose:
    - Handles single-city countries where country-wise mean = single value
    - Ensures NO missing values in final dataset

Result:
  ✓ Zero missing values after this step
  ✓ All 15 numerical features are complete
  ✓ Ready for scaling and encoding


SECTION 6: OUTLIER DETECTION (IQR METHOD)
────────────────────────────────────────────────────────────────────────────────
Purpose: Identify extreme values (outliers) in each numerical feature

Method: Interquartile Range (IQR) - Standard Statistical Approach

Calculation for each feature:
  1. Q1 = 25th percentile
  2. Q3 = 75th percentile
  3. IQR = Q3 - Q1
  4. Lower Bound = Q1 - 1.5 × IQR
  5. Upper Bound = Q3 + 1.5 × IQR
  6. Outliers = values < Lower Bound OR > Upper Bound

Output:
  • Count of outliers per feature
  • Percentage of outliers
  • Min and max values for validation

Decision Applied: KEEP OUTLIERS AS-IS
  Rationale:
    - Outliers represent real-world extremes (very safe cities, expensive areas)
    - These extremes are valuable for the model to learn city characteristics
    - Removal would lose important information about city diversity
    - Examples: Zurich (expensive), Luxembourg (small), etc.

Visualization:
  • Box plots for all 15 features (3×5 grid)
  • Red titles indicate features with detected outliers
  • Helps visually inspect outlier patterns


SECTION 7: CATEGORICAL ENCODING (LABEL ENCODING)
────────────────────────────────────────────────────────────────────────────────
Purpose: Convert categorical Country column to numerical format for ML models

Method: Label Encoding (Ordinal/Integer Encoding)

Process:
  1. Extract unique countries from dataset
  2. Assign each country a unique integer (0, 1, 2, ...)
  3. Create "Country_Encoded" column with integer values
  4. Sort countries alphabetically for consistency

Encoding Scheme:
  • Austria → 0
  • Belgium → 1
  • Canada → 2
  ... (sorted alphabetically)
  • Total: 16 unique countries

Output:
  • New column "Country_Encoded" with integer values (0-15)
  • Country mapping table for reference
  • Original "Country" column retained for EDA/grouping

Why Label Encoding (not One-Hot)?
  - Smaller memory footprint (1 column vs. 16 columns)
  - Simpler implementation
  - Works well with tree-based models that handle ordinal features
  - Sufficient for this use case with <20 unique values


SECTION 8: SCALE NUMERICAL FEATURES (STANDARDSCALER)
────────────────────────────────────────────────────────────────────────────────
Purpose: Normalize all numerical features to have mean=0 and std=1

Method: StandardScaler (Z-score Normalization)

Formula for each feature value x:
  x_scaled = (x - mean) / standard_deviation

Applied to ALL 15 numerical features:
  • 9 core features (already complete)
  • 6 sparse features (already imputed)

Scaling Process:
  1. Calculate mean and std for each feature
  2. Subtract mean from each value (centering)
  3. Divide by standard deviation (scaling)

Benefits:
  • Puts all features on same scale
  • Prevents features with large values from dominating models
  • Improves convergence in distance-based algorithms
  • Required for most ML algorithms (SVM, KNN, Neural Networks, etc.)

Verification:
  • Mean ≈ 0 (within machine precision, ~1e-8)
  • Std ≈ 1 (within machine precision, ~1e-8)
  • Individual feature statistics displayed for validation
