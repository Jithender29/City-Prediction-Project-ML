7.3 Hyperparameter Tuning
------------------------------------
Method: GridSearchCV with 5-Fold Cross-Validation
Scoring Metric: R² (coefficient of determination)

Gradient Boosting Tuning Details:
  
  Learning Rate: [0.03, 0.05, 0.1]
    - Controls step size of error correction
    - Lower values: Better generalization, slower convergence
    - Higher values: Faster fitting, risk of overfitting
    - Optimal: 0.1 (fast convergence, still generalizes well)
  
  Number of Estimators: [100, 200, 400]
    - Number of sequential trees to train
    - More estimators: Better fit, longer training
    - Optimal: 400 (most trees in best model)
  
  Max Depth: [2, 3, 4]
    - Depth of individual trees (shallow for boosting)
    - Shallow trees: Better generalization (bias-variance tradeoff)
    - Optimal: 2 (very shallow, prevents overfitting)
  
  Subsample: [0.8, 1.0]
    - Fraction of samples used per boosting iteration
    - < 1.0: Adds stochasticity, regularization effect
    - Optimal: 0.8 (uses 80% of data per iteration)
  
  Min Samples Leaf: [1, 2]
    - Minimum samples required in leaf node
    - Higher values: Simpler trees, less overfitting
    - Optimal: 1 (allow single-sample leaves in shallow trees)

Training Process:
  1. Create parameter grid with all combinations (216 total)
  2. For each combination:
     a. Initialize GradientBoostingRegressor with parameters
     b. Perform 5-Fold Cross-Validation on training data (90 samples)
     c. Record mean CV R² and standard deviation
  3. Select best combination based on validation performance
  4. Retrain on combined train+validation set (120 samples)
  5. Evaluate on held-out test set (30 samples)

Result: Best of 216 configurations selected
  Top Configuration: learning_rate=0.1, n_estimators=400, max_depth=2, subsample=0.8, min_samples_leaf=1
  CV R²: 0.9449 ± 0.0169 (standard deviation across 5 folds)

7.4 Model Evaluation Metrics
------------------------------------
Primary Metric: R² (Coefficient of Determination)
  Formula: R² = 1 - (SS_res / SS_tot)
  Interpretation: Proportion of variance explained by model
  Range: 0 to 1 (higher is better)
  
  SS_res = Σ(y_actual - y_predicted)²
  SS_tot = Σ(y_actual - y_mean)²

Secondary Metric: MAE (Mean Absolute Error)
  Formula: MAE = (1/n) × Σ|y_actual - y_predicted|
  Interpretation: Average absolute prediction error
  Units: Same as target (Livability Score points, range 1-150)
  Advantage: Interpretable and robust to outliers

Training Metrics (on Training Set - 90 samples):
  Implicit through cross-validation during GridSearchCV

Validation Metrics (on Validation Set - 30 samples):
  - Validation R²: 0.9387
  - Validation MAE: 8.234 (average error ~8 livability points)
  
  Used to: Select best model from hyperparameter candidates
  Purpose: Avoid test set contamination

Test Metrics (on Test Set - 30 samples):
  - Test R²: 0.9227 (92.27% variance explained)
  - Test MAE: 9.857 (average prediction error ~9.86 points)
  
  Interpretation:
    - Model explains 92.27% of variation in test livability scores
    - Average prediction error is 9.86 points on 1-150 scale
    - Acceptable performance for city livability prediction
    - Validates model generalization to unseen cities

Cross-Validation Results (5-Fold CV on Training Set):
  - Mean CV R²: 0.9449
  - Std Dev: 0.0169 (low variance = stable model)
  - Fold Results: Consistent performance across all 5 folds
  
  Interpretation:
    - Model performs reliably on different data subsets
    - Low standard deviation indicates robust performance
    - No evidence of extreme overfitting or underfitting

Performance Progression:

9.2 Evaluation Metrics Analysis
------------------------------------
Primary Metric: R² (Coefficient of Determination)

Definition: Proportion of variance in target explained by model
Formula: R² = 1 - (SS_res / SS_tot)
  where SS_res = Σ(actual - predicted)²
        SS_tot = Σ(actual - mean)²

Interpretation for This Model:
  - R² = 0.9227 means:
    "Model explains 92.27% of variation in city livability scores"
  - The remaining 7.73% unexplained variation due to:
    * Model limitations (feature set may be incomplete)
    * Data noise and measurement error
    * Unmeasured factors affecting livability
    * Inherent unpredictability in human perception of livability

Performance Benchmarks:
  R² < 0.5: Poor model
  R² 0.5-0.7: Fair model
  R² 0.7-0.85: Good model
  R² 0.85-0.95: Excellent model ← OUR MODEL
  R² > 0.95: Outstanding model (rare in practice)

Secondary Metric: MAE (Mean Absolute Error)

Definition: Average absolute deviation between predicted and actual
Formula: MAE = (1/n) × Σ|actual - predicted|
Units: Points on Livability Score scale (1-150)

Test Set MAE: 9.857 points

Interpretation:
  - Average prediction error: ±9.86 livability points
  - Relative error: 9.86 / (150 - 1) = 6.6% of scale range
  - On average, model predictions are off by ~9.86 points
  - For 70% of predictions, error likely <15 points
  - For 90% of predictions, error likely <20 points

Advantages of MAE:
  - Interpretable in original units
  - Robust to outliers (unlike MSE/RMSE)
  - Linear penalty for errors (unlike squared errors)

Advantages of R²:
  - Scale-independent (dimensionless 0-1 range)
  - Directly shows proportion of variance explained
  - Standard for regression model comparison

Cross-Validation Stability:
  Mean: 0.9449
  Std Dev: 0.0169
  Coefficient of Variation: 0.0169 / 0.9449 = 0.18%
  
  Interpretation:
    - Very low variation across folds (<2%)
    - Model performs consistently on different data subsets
    - Not dependent on specific training split
    - Robust and stable performance
